Git Forensics MCP Server

MCP server in Rust that exposes git repository analysis tools over stdio.
An MCP-compatible AI client connects to this server and calls tools to analyze a local repo.

This is a learning project — the goal is to understand MCP server patterns in Rust,
not just have working code. Teach concepts, don't just write code.

Tools:
  - blame: show who last touched each line of a file
  - file_history: commit log for a specific file
  - hotspots: files that change most often (churn analysis)

Stack:
  - rmcp: MCP SDK (server + stdio transport)
  - git2: libgit2 bindings for reading git repos
  - tokio: async runtime
  - serde / schemars: serialization + JSON schema generation for tool inputs
  - anyhow: error handling
  - tracing: structured logging (to stderr, since stdout is the JSON-RPC transport)

Build order:
  1. Dependencies & project setup (Cargo.toml)
  2. Server struct (GitForensicsServer) + empty main with stderr logging + stdio transport
  3. ServerHandler trait — get_info() with #[tool_handler]
  4. First tool: blame (BlameRequest struct, #[tool] function, spawn_blocking)
  5. Second tool: file_history (revwalk API, pathspec filtering)
  6. Third tool: hotspots (HashMap accumulation, sorting, take(N))
  7. Wire up to Claude Desktop and test on a real repo

Key concepts:
  - stdout is the JSON-RPC transport — never println!, always log to stderr
  - git2::Repository is !Send (libgit2 uses thread-local state), so all git work
    goes inside spawn_blocking. Clone what you need, move it in, return a result.
  - #[tool_router] on the impl block scans for #[tool] methods and generates
    Self::tool_router(). #[tool_handler] on the ServerHandler impl wires that
    router into handle_request.
  - Parameters<T> is an extractor — when the AI calls a tool with JSON args
    like {"path": "src/main.rs"}, rmcp deserializes it into your request struct.
    The JsonSchema derive generates the schema the AI client sees.
  - CallToolResult::success wraps output in MCP content format for the AI.

Testing with Claude Desktop:
  cargo build --release, then add to:
  ~/Library/Application Support/Claude/claude_desktop_config.json

  {
    "mcpServers": {
      "git-forensics": {
        "command": "/path/to/target/release/mcp-test",
        "args": ["/path/to/some/git/repo"]
      }
    }
  }
